> 日期：2026-06-24　｜　阶段：里程碑 2「能打」

## 1. 这一步做什么 / 不做什么（范围）

**做：**
- 第一人称**红龙步枪**（代码搭的卡通模型，挂相机右下角）

- **打中判定**（射线 hitscan）：复用现成的 `rayBoxEntry`；木头人分**头/身**两块；**爆头一枪秒**
- **准度**：站定准、移动飘、跳起最飘、蹲下更稳、连发越扫越散
- **蛋蛋木头人**：满血 100、受击闪白 + 火花、打死倒地、**3 秒满血复活**
- **弹匣 25 发**，按 **R** 换弹（约 2 秒）、打空自动换、空仓"咔哒"
- **真实音效**（CC0/CC-BY，记授权）：枪声、换弹、打中(身/墙)、爆头确认
- **HUD**：右下子弹数、**动态准星**（飘就张开）、**命中闪标记**(hitmarker)

## 3. 准度模型（站定准 / 移动飘）——纯逻辑、可测

一个**散布角** `spread`（子弹偏离准星中心的最大角度）。开火时在以准星为中心、张角 = `spread` 的圆锥里**随机**取一个方向射出。`spread` 由当前状态叠加：

- **基础**（站定不动、不连发）：≈0.2°（基本指哪打哪）
- **移动**：加一档，跟水平速度成正比（走路加得多，**静步只加一点点**——奖励慢慢挪）
- **在空中**（跳）：加一大档（几乎打不准）
- **蹲下**：× 0.6（更稳）
- **连发热度** bloom：每开一发 +一点，随时间冷却；**扫射越久越散**，停一下收回来

> 注：疾跑已删，所以没有"疾跑飘"这种状态，正好少一档、简单。

→ 抽成纯函数 `computeSpread({speed, airborne, crouch, heat}): 角度`，写在 `weapons/spread.ts`，**可单元测试**。

## 4. 打中判定（hitscan 射线）——纯逻辑、可测

1. 从**相机位置**出发，方向 = 准星朝向按 `spread` 随机偏一点，射一条射线（最远 ~200 米）。
2. 候选命中：
   - 每个**活着**的木头人 = **头盒** + **身盒** 两个 AABB（跟着它的位置走）
   - 所有**墙/掩体** = 地图已有的 `walls`
3. 复用 `rayBoxEntry`（菜单战斗 `attractBattle.ts` 里已有、用过）——把它**抽到公用** `weapons/ray.ts`，菜单和射击都用，去重。
4. 求**最近**命中点（比所有 AABB 的入射参数 t，取最小）：
   - 最近是某木头人**头盒** → **爆头**（-160，命中点）
   - 是**身盒** → 身体（-36）
   - 是**墙** → 没打中人（弹道停墙上，冒灰火花）
5. 返回 `{ kind: 'head' | 'body' | 'wall' | 'none', point, dummy? }`

> 关键：墙挡在木头人前面时，先碰墙 → 打不中人（躲掩体有效）。这点要写测试。

## 5. 蛋蛋木头人（`enemies/dummy.ts`）

- **外形**：复用 `createEgg()`（蛋身 + 帽子 + 豆豆眼 + 小枪），用**敌方蓝色**当靶子；可选脚下加个小"靶环"标记。**站着不动、不还手**（不是 AI，纯靶子）。
- **数据**：`hp`、`alive`、头盒 + 身盒、复活计时。蛋身实际范围约 x/z ±0.5、y 0~1.3（帽顶）。划分：**头盒**＝上面一截（约 y 0.85~1.3，含脸和帽子）；**身盒**＝下面（约 y 0~0.85）。两盒跟着木头人位置走。
- **受击**：`hp -= dmg`；**闪白** 0.1 秒；命中点冒火花；`hp<=0` → 死。
- **死**：往后**倒地**（绕脚边轴翻倒的小动画）+ 不可再被打中；**3 秒**后站起、`hp` 回满。
- **摆位**：沙漠图上摆 **6~8 个**（中路、A 点、B 点、远近不同距离），位置在 `main` 或单独 `dummies` 配置里给。

## 6. 第一人称枪模型（`weapons/viewmodel.ts`）

- 代码搭一把**卡通红龙**：枪身/握把/弹匣/枪管用几个盒子 + 圆柱拼，统一卡通配色（深枪身 + 一点亮色点缀），跟蛋蛋一个风格。
- 挂在**相机的子物体**上，固定在**右下角**；有轻微"待机摇摆"，开火后**往后弹一下再回弹**。
- **枪口节点 `muzzle`**：火光从这里冒、弹道线从这里画到命中点。
- 开火视觉：枪口**火光**（发光面片，亮一帧）+ 枪身后坐 + **弹道线**（复用 `tracer` 那套淡出线段）。

> 接口留好：以后想换"下载的精致 GLTF 红龙"，只要替换这个模块、保持 `muzzle` 节点即可。

## 7. 射击控制（`weapons/shooting.ts`）——驱动一切

- 读输入：**左键按住** = 想开火；**R** = 换弹。
- 状态机：`ready`（可开火）/ `firing`（按射速节流，每 0.1 秒一发）/ `reloading`（2 秒）/ `empty`（空仓）。
- **每发**做这串事：扣 1 发子弹 → 算 `spread` → 射线判定 → 给木头人结算伤害 → 视觉（火光/弹道/后坐）→ 音效 → 更新 HUD。
- 子弹 = 0 且想开火 → 空仓"咔哒" + **自动换弹**；换弹中不能开火。
- 依赖：`input` 加左键(fire)/R(reload)；`playerController` 暴露 `speed / airborne / crouch` 给 `spread` 用。

## 8. 声音（`audio/sfx.ts`）——真实素材，记授权

- 用 Three.js 音频：
  - **自己的枪声、换弹**：普通 `Audio`（贴脸，不分远近）
  - **打中木头人 / 打墙**：`PositionalAudio`（放命中点，有远近大小）
- 需要的音效（**全部 CC0 / CC-BY**，逐个记进 `docs/CREDITS.md`）：
  1. 步枪**单发枪声**（连发就快速重复播这一个）
  2. **换弹声** + 空仓"**咔哒**"
  3. 打中**身体**"噗/咚" + 打中**墙**"哒"
  4. **爆头确认**"叮"
- 来源优先级：**Kenney**（CC0）→ **OpenGameArt**（CC0）→ **freesound**（CC0 或 CC-BY 署名）。**已确认本机能联网下载**。
- 兜底规则：万一找不到合适的**真实**音效，宁可先**空着**并在 CREDITS 记 TODO，**绝不用搞笑/合成音糊弄**（守要求 #4、#5）。

## 9. HUD（`index.html` + UI 逻辑）

- **右下**：子弹数 `25 / 25`（当前/弹匣）；换弹时显示"换弹中…"。
- **准星**：动态——`spread` 越大越**张开**（4 条小线或一个圈，随 `spread` 缩放）；命中时**闪一下**(hitmarker)，**爆头**变特殊色（如黄/红）。
- 这些都是纯展示，从 `shooting` 的状态读数。

## 12. 帧率（守 ≥ 30、争 60）

- 弹道线、火花、音效用**对象池或复用**（别每发 `new`）。
- 木头人数量少（6~8 个）。
- 射线判定是纯数学，很便宜。
- 枪口火光用一个反复开关的发光面片，不新建。

## 14. 完成标准（Definition of Done）

- 能拿红龙在沙漠图上连发，打中木头人掉血、**爆头一枪秒**、墙后安全。
- 站定准、移动飘、蹲下更稳、连发越扫越散——手感对。
- 弹匣 25、R 换弹、打空自动换，子弹 HUD 正确。
- 枪声/换弹/打中/爆头**真实音效**都响，授权记进 CREDITS。
- 动态准星 + 命中闪标记。
- 自动测试全过，手动试玩清单逐条通过，帧率 ≥ 30，无明显 bug。
