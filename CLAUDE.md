# 爆蛋行动（项目代号 CMGO）—— 我们自己的射击游戏项目

> 一句话：用网页技术，做一个**卡通风格、好看、好玩、质量高**的第一人称射击游戏《**爆蛋行动**》，**玩法像《CS》**（回合制爆破、买"爆蛋器"、两队对抗），以后还能联机，最后发布到 GitHub，朋友点开网址就能玩。

- **游戏名**：《爆蛋行动》
- **炸弹叫**：爆蛋器（相当于 CS 里的 C4 炸弹）

## 2. 玩法（像 CS 那样玩）

我们要做 CS 最经典的玩法 —— **回合制"爆破模式"**：

- **两队对抗**：
  - **进攻队**：把**爆蛋器**（就是炸弹）安到包点上，并保护它直到爆炸。
  - **防守队**：阻止对方安爆蛋器；如果被安上了，就在爆炸前**拆掉**它。
- **怎么算赢一个回合**（做到一个就赢）：爆蛋器爆炸 ✓ / 成功拆除 ✓ / 把对方全消灭 ✓。
- **回合制**：一局有很多回合，先赢够规定回合数的队伍获胜。
- **死了要等**：一个回合里被打死，就先**观战**，等下一回合再上（这是 CS 的特点）。
- **买枪 + 钱**：每回合开始有一点钱，用来买**枪、护甲、手雷**；赢、输、击杀都会影响下回合的钱。
- **武器**：刀、手枪、步枪、狙击枪，还有手雷（高爆 / 闪光 / 烟雾）。
- **打枪手感（很关键）**：能走、跑、跳、蹲；**站着不动打得准，边跑边打打不准**；**爆头**伤害特别高。
- **地图**：有两个包点(A/B)、各自出生点、中路和掩体 —— 但**用我们自己做的地图和名字**，不直接抄 CS 的（避免版权问题）。

> 💡 CS 完整玩法其实很多，我们**先做核心**（回合 + 爆蛋器 + 买枪 + 射击手感），细节一点点补，慢慢就越来越像了。

## 3. 技术选择（我们用什么做）

| 用途 | 选择 | 为什么 |
|------|------|--------|
| 编程语言 | **TypeScript** | 写代码时能自动帮我们抓很多 bug（对应要求 #8）。 |
| 打包/启动工具 | **Vite** | 启动快，改代码网页自动刷新，方便看效果。 |
| 3D 画面引擎 | **Three.js** | 网页里做 3D、卡通光照、加载精美模型、放声音都很在行。 |
| 碰撞/物理 | 先自己写**简单碰撞**，需要时再加 `cannon-es` | 先简单后复杂，避免一上来太难。 |
| 声音 | Three.js `PositionalAudio` | 能做"离得近声音大、离得远声音小"的真实效果。 |
| 多人联机（以后） | **WebSocket** + 一个小 Node 服务器 | 等单机做好了再加。 |

> **为什么选网页版**：打开网址就能玩、不用下载、最容易发到 GitHub 分享。

## 6. 开发计划（分阶段，一步一步来）

> 规则：**做完一个阶段、试玩通过、提交到 git，再开始下一个。**
> 质量规则：**每个阶段都要用精美的卡通模型 + 好光影，不把"好看"留到最后。**

- **阶段 0｜搭骨架**：项目能 `npm run dev` 跑起来，网页上出现一个 3D 场景（先一个地板 + 一个临时方块，只为确认能跑）。
- **阶段 1｜能动 + 像样的场景**：第一人称视角，能用键盘走、跑、跳、蹲，用鼠标看；有地板和墙，撞墙走不过去（碰撞）；**这一步就换上精美的卡通模型和好光影**，让场景一开始就好看。
- **阶段 2｜能打**：拿一把**精美的卡通枪模型**，能开枪、有子弹、能打中目标；有准星、枪口火光、真实射击音效；做出**爆头伤害高**、**移动时打不准 / 站定才准**的手感。
- **阶段 3｜爆破玩法核心**：
  - 一个完整回合：能**安爆蛋器 / 拆爆蛋器**，按"爆蛋器炸了 / 拆掉 / 全消灭"判断这回合谁赢。
  - 电脑机器人敌人(AI)：用好看的卡通人物模型，会巡逻、追玩家、开枪、躲掩体。
  - 一张做好的地图（合理布局、精美场景、A/B 包点）。
- **阶段 4｜完整一局（像 CS）**：加**买枪菜单 + 经济（钱）+ 多回合 + 护甲 / 手雷 + 计分板 + 输赢判定**。做完就是一个**好看又好玩的完整单机《爆蛋行动》**了。
- **阶段 5｜多人联机**：加服务器，让两个人能一起玩。
- **贯穿始终**：找精美卡通素材、优化帧率、查 bug、定期发到 GitHub。

## 7. 美术 & 音效素材规则

- **只用免费、授权清楚的精美素材**，优先**卡通 / 低多边形**风格。推荐来源：
  - 模型：Kenney.nl（CC0 免费，本身就是卡通风，很合适）、Quaternius、Poly Pizza、Sketchfab（注意看授权）。
  - 音效：freesound.org、Kenney 音效包、OpenGameArt。
- 选模型时优先挑**卡通风格统一、做工精细**的，保证整体好看。
- 每用一个素材，就在 `docs/CREDITS.md` 里**记下来源和授权**（防止以后侵权）。
- **绝不**用有版权问题的素材；**绝不**用表情包 / 梗图（对应要求 #5）；**不直接抄** CS 的官方地图和素材。

## 8. 光照怎么做（给 Claude 的技术提示）

> 我们是**卡通画面 + 真实光影**（像《堡垒之夜》）：画面卡通，但阴影和明暗照样做好，更耐看。

- 材质用 `MeshStandardMaterial`（PBR），卡通模型也能有漂亮的光影。
- 用一盏 `DirectionalLight` 当太阳 + 一点 `HemisphereLight` 补光。
- 打开阴影：`renderer.shadowMap`，用 `PCFSoftShadowMap`（柔和阴影）。
- 色调映射用 `ACESFilmicToneMapping`，画面更通透好看。
- 用环境贴图（环境光反射）让物体更有质感。
- ⚠️ 灯多、阴影多会**变卡**——要和"帧率 ≥ 30fps"这条平衡，别贪多。

---

## 9. 帧率怎么保证 ≥ 30fps

- 控制模型面数（卡通 / 低多边形模型本来就轻，很合适）。
- 子弹等大量重复的东西用**对象池**（重复利用，别一直新建）。
- 限制会投影的灯的数量。
- 限制 `renderer.pixelRatio`（一般不超过 2）。
- 用 `stats.js` 实时看帧率，**在普通笔记本上**测，不只看高端电脑。

---

## 10. 少 bug 的做法（对应要求 #8）

- TypeScript 开**严格模式**，让它帮我们提前发现错误。
- 用 ESLint 检查代码、Vitest 给游戏逻辑（如伤害计算、回合判定、AI 状态、经济算钱）写自动测试。
- 每个阶段结束前，按**试玩清单**手动玩一遍（走/跑/跳/蹲/射击/爆头/安爆蛋器/拆爆蛋器/买枪/输赢都试到）。
- **说"做好了"之前，一定先真的跑一遍、看到它正常，才能说成功。**
